## Implemented Features

### 1. Tailwind CSS v3 Integration
- Successfully set up and configured **Tailwind CSS v3**.
- Verified functionality using utility classes like `bg-green-100`, `text-cyan-400`, etc.
- Customized `tailwind.config.js` for future scalability and color theme control.

### 2. Dark/Light Mode Toggle
- Implemented a **theme toggle** using Lucide icons (`<Sun />`, `<Moon />`).
- Theme preference is saved in `localStorage` and respects system settings on first load.
- `dark` class is applied to `document.documentElement` dynamically.
- Tailwind-based styles reflect mode across all components.

### 3. Responsive Header with Burger Menu
- Built a sticky, mobile-responsive **Header** with `<Shield />` icon and page links.
- Mobile menu toggles with `<Menu />` and `<X />` icons using a smooth transition.
- Desktop nav hidden on small screens, visible with `md:flex`.

### 4. Modern UI Layout (Grid System)
- Used TailwindвЂ™s `grid-cols-1 sm:grid-cols-2 lg:grid-cols-3` layout to render article cards.
- Optimized spacing with `gap-6`, `mt-20`, and container alignment.

### 5. Dynamic Content Fetching
- Used `useEffect` and native `fetch` to load articles from `/api/reports`.
- Handled **loading**, **error**, and **empty** states with user-friendly UI messages.
- Displays total number of articles at the bottom of the grid.

### 6. Stylized Article Cards
- Built `ReportCard.jsx`:
  - Displays title, summary, analysis, and optional full content.
  - Icons dynamically chosen based on article type (`ai`, `network`, `ransomware`...).
  - Background color of risk badge reflects severity (critical, high, medium, low, unknown).
  - Uses `line-clamp`, transitions, and border effects for a clean visual look.
